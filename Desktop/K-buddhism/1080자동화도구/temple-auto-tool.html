🔄 카톡 → 홈피 전체 작업 흐름

    

      

        

        
📨 사찰로부터 카톡 수신

        
위 안내문을 받은 사찰이 내용 + 사진을 카톡으로 전송
사진은 카톡 앨범으로 받거나 링크로 공유받기

        
소요시간: 수신 즉시카카오톡 채널

      

      

        

        
🖼️ 사진 → Cloudinary 업로드

        

          1. cloudinary.com 접속 → Media Library

          2. 사진 파일 드래그&드롭 업로드

          3. 업로드된 사진의 Public ID 복사 (파일명 형태)
        

        
소요시간: 5~10분Cloudinary 무료 플랜

      

      

        

        
📝 이 도구에 입력 → MD 생성

        

          "카톡→MD 변환기" 탭에서 정보 입력

          MD 파일 자동 생성 버튼 → 파일 다운로드
        

        
소요시간: 10~15분이 도구

      

      

        

        
🚀 GitHub 업로드 → 자동 배포

        

          MD 파일을 src/content/temples/ 폴더에 넣고

          터미널에서 git add . → git commit → git push

          → Vercel 자동 배포 (약 30초) → 홈피 완성!
        

        
소요시간: 2~3분GitHub + Vercel

      

    

    

      ⏱️ 1사찰 총 소요시간: 약 20~30분 (숙달되면 15분 이내)

      🎯 하루 8사찰 × 135일 = 1,080사찰 완성 가능
    

  


  

    
💡 Cloudinary Public ID 찾는 법

    

      1단계 → cloudinary.com 로그인

      2단계 → 왼쪽 메뉴 "Media Library" 클릭

      3단계 → 업로드한 사진 위에 마우스 올리기

      4단계 → "Copy URL" 또는 사진 클릭 → 오른쪽 패널에서 Public ID 확인

      

      📌 예시: haeinsa_main_bcd789 이 부분만 복사

          • Python 스크립트 1개로 사진 일괄 업로드

          • Public ID 목록 자동 생성 (복사→붙여넣기)

          • 사찰이 카톡 채널에 지정 형식으로 입력

          • 챗봇이 자동으로 Google Sheets에 저장

📋 GitHub Desktop 사용법 (터미널 없이 push)

    

      💡 터미널(검은 창)이 어려우시면 GitHub Desktop 앱을 사용하세요. 클릭만으로 됩니다!
    

    

      1. desktop.github.com 에서 GitHub Desktop 설치

      2. K-buddhism 저장소 연결 (File → Open Repository)

      3. MD 파일을 src/content/temples/ 폴더에 복사

      4. GitHub Desktop 앱에서 변경 파일 자동 감지 확인

      5. 하단 Summary에 "새 사찰: 해인사" 입력

      6. "Commit to main" 버튼 클릭

      7. "Push origin" 버튼 클릭

      8. Vercel 대시보드에서 30초 후 자동 배포 확인 ✅
